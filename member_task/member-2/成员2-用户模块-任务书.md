**注册流程**：
```
① 前端传参：phone, code, password, studentId, realName
② 校验验证码（固定"888888"）
③ 校验手机号唯一 → user表phone字段查重
④ 校验学号格式（正则）+ 学号唯一
⑤ BCrypt加密密码
⑥ 创建user记录 + user_auth记录（auth_status=已认证）
⑦ 调用成员1的JwtUtils生成token返回
⑧ 返回：{ token, userId, nickname }
```

**登录流程**：
```
① 前端传参：phone, password
② 查user表 by phone
③ BCrypt密码比对
④ 生成JWT token返回
```

## 关键提醒

- **密码必须BCrypt加密**，绝对不能明文存储
- **注册时校验手机号和学号的唯一性**，用数据库唯一索引 + Service层捕获异常双重保障
- **短信验证码在MVP阶段固定为"888888"**，前端做60秒倒计时即可
- **学号校验用正则**，如学校格式为"20XX10XXXX"，用RegexUtils
- **地址管理**是校园特色的体现，校区/楼栋/宿舍号字段要保留，前端用级联选择器
- **注册页的分步表单**每步都要做前端校验，密码确认必须一致
